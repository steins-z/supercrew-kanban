## Problem Statement

Currently, the kanban board only displays `.supercrew/features/` data from the `main` branch. Developers working on feature branches cannot see their changes reflected in the kanban until they merge to main. This creates a visibility gap for work-in-progress features.

## Design Draft

### Solution Overview

Implement a multi-branch kanban that scans all `main + feature/*` branches, intelligently deduplicates identical content, and displays cards with branch tags for near-realtime visibility of all development work.

### Core Algorithm

**Diff & Deduplication:**
1. Fetch all branches: `GET /repos/{owner}/{repo}/git/refs/heads/feature`
2. Parallel fetch features from each branch using `Promise.allSettled`
3. For each feature, fetch `meta.yaml`, `design.md`, `plan.md` with `?ref={branch}`
4. Compute MD5 hash: `hash = md5(meta + design + plan)`
5. Group by feature ID, then by hash
6. Build cards: same hash → single card with multiple branch tags

**Example:**
```
feature-oauth:
  hash_abc: [main, feature/ui]      → 1 card, tags: [main, ui]
  hash_xyz: [feature/oauth-backend] → 1 card, tag: [oauth-backend]
```

### Performance & Error Handling

**Caching:**
- Simple in-memory cache with 30s TTL
- Vercel serverless: cold start loses cache, hot requests < 100ms

**Error Handling:**
- Use `Promise.allSettled` for partial failure tolerance
- Continue with available branches if some fail
- Display warning banner for failed branches
- Track GitHub rate limit (5000/hour)

**Response Format:**
```json
{
  "features": [...],
  "featuresByStatus": {...},
  "metadata": {
    "scannedBranches": ["main", "feature/oauth"],
    "totalBranches": 5,
    "fetchedAt": "2026-03-05T10:00:00Z",
    "errors": [
      { "branch": "feature/old", "error": "404", "type": "not_found" }
    ]
  }
}
```

### Out of Scope (v1)
- Editing features from non-main branches
- Branch selection UI (scans all feature/* automatically)
- Real-time webhooks (polling only)
- Redis/Vercel KV caching
