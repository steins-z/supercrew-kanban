# Kanban Column Scrolling

## Background

目前看板界面中，当某一列的任务卡片过多时，列会无限向下延伸，导致用户无法在屏幕内看到所有卡片。用户需要滚动整个页面才能查看下方的卡片，这种体验很不友好。

理想的看板体验应该是：
- 每一列独立可滚动
- 列的高度固定在视口范围内
- 用户可以在列内部滚动查看所有卡片
- 不需要滚动整个页面

这个问题在有 10+ 个任务的列中尤为明显，严重影响了用户的使用体验和工作效率。

## Requirements

### 核心功能

1. **列内滚动**
   - 为每个看板列的卡片容器添加垂直滚动功能
   - 设置合理的 `maxHeight` 限制列的高度
   - 使用 `overflowY: auto` 实现自动滚动条

2. **高度计算**
   - 列的最大高度应该考虑：
     - 顶部 AppHeader 的高度
     - 底部 Dock 的高度
     - 列标题的高度
     - 适当的内边距和间距
   - 建议使用 `calc(100vh - 220px)` 或类似的动态计算

3. **保持现有布局**
   - 列的宽度保持不变 (264px)
   - 水平滚动功能保持不变
   - 列标题固定在顶部，不随内容滚动
   - 卡片之间的间距和动画效果保持不变

4. **滚动条样式**
   - 使用系统原生滚动条或自定义样式
   - 确保在深色/浅色主题下都清晰可见
   - 滚动条不应占用过多空间

### 用户体验

- 滚动应该流畅，没有卡顿
- 滚动条只在内容超出时显示
- 鼠标悬停在列上时，滚动应该在该列内部进行，而不是整个页面
- 支持鼠标滚轮、触摸板、拖动滚动条等多种滚动方式

## Out of Scope

以下功能不在本次实现范围内：

- 虚拟滚动（virtualization）优化性能
- 无限滚动/懒加载
- 列的折叠/展开功能
- 卡片的拖拽重排（当前是只读看板）
- 响应式布局调整（移动端优化）
- 横向滚动的改进
